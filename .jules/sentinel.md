## 2024-05-20 - IP Spoofing Bypass in Rate Limiting
**Vulnerability:** The rate limiter in `src/app/api/contact/route.ts` extracts the client IP address from the *first* value in the `x-forwarded-for` header chain. This allows an attacker to bypass the rate limit by prepending a fake IP address to the `x-forwarded-for` header.
**Learning:** In environments behind reverse proxies (like Vercel or AWS API Gateway), the true client IP is appended to the *end* of the `x-forwarded-for` chain by the proxy nearest to the server. The first value is user-controlled and easily spoofable.
**Prevention:** Always extract the *last* IP address from the `x-forwarded-for` chain, or better yet, rely on platform-specific headers guaranteed by the hosting provider (e.g., `x-vercel-forwarded-for`, `x-real-ip` when set securely by proxy).
